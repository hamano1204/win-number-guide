# WinNumberGuide 設計書

## 1. プロジェクト概要

**WinNumberGuide** は、Windows 11（およびWindows 10）環境において、Windowsキーを長押しした際にタスクバーにピン留め・起動されているアプリケーションのショートカット番号（1〜9、0）を画面中央にオーバーレイ表示するデスクトップアプリケーションです。
ユーザーは視覚的に「Winキー + 数字キー」の対応を確認でき、アプリケーションの切り替えや起動をスムーズに行うことができます。

## 2. システム要件・技術スタック

- **対象OS**: Windows 10 / Windows 11
- **フレームワーク**: .NET 10.0 (net10.0-windows10.0.22621.0)
- **UIテクノロジー**: WPF (Windows Presentation Foundation)
- **主要なWindows API・ライブラリ**:
  - `UIAutomationClient` (タスクバー情報の取得)
  - `user32.dll` (グローバルキーボードフック用ネイティブAPI)
  - `System.Windows.Forms.NotifyIcon` (システムトレイアイコン)
  - `Microsoft.Win32.Registry` (スタートアップ登録)

## 3. アーキテクチャ・コンポーネント構成

本アプリケーションは、主に以下のコンポーネントから構成されています。

### 3.1. キーボード入力監視 (KeyboardHook.cs)
低レベルキーボードフック（`WH_KEYBOARD_LL`）を使用し、OS全体のキーボードイベントを監視します。
- Windowsキー（左/右）の押下状態を追跡します。
- Winキーが単独で押され、一定時間（900ms）保持された場合に `WinKeyLongPressed` イベントを発火します。
- 他のキーが押された場合や、Winキーが離された場合はタイマーをキャンセルし、UIを非表示にするためのイベントを発火します。

### 3.2. タスクバー情報取得 (TaskbarReader.cs)
Windowsの「UI Automation」を利用して、エクスプローラー（タスクバー）のUI要素ツリーを解析します。
- クラス名 `Shell_TrayWnd` のルート要素から、タスクバーボタン（`Taskbar.TaskListButtonAutomationPeer`）を検索します。
- 各ボタンからアプリケーション名、AutomationId（AppId）を取得します。
- 取得した情報を元に、左から順に番号（1〜9, 0）を割り当て、表示用のデータモデル（`TaskbarApp`）を生成します。左右のキーボード配置を考慮し、5番目と6番目の間に余白（マージン）を挿入するロジックも含みます。

### 3.3. アイコン抽出 (IconExtractor.cs / PackagedAppIconExtractor.cs)
タスクバー上の各アプリケーションから高解像度のアイコンを抽出します。
通常のWin32アプリケーションのアイコンだけでなく、UWP（パッケージ化された）アプリのアイコン抽出にも対応するよう設計されています。

### 3.4. ユーザーインターフェース (MainWindow.xaml / MainWindow.xaml.cs)
取得したタスクバーアプリケーションのリストを画面に表示するWPFウィンドウです。
- **オーバーレイ表示**: 画面の中央に配置され、透過とアニメーション（フェードイン・フェードアウト）を用いて自然に表示・非表示を切り替えます。
- **タスクトレイ常駐**: ウィンドウ自体はバックグラウンドで動作し、システムトレイ（通知領域）のアイコンから「スタートアップ登録」や「終了」などの操作を行います。
- **メモリ管理**: UIが非表示になる際（フェードアウト完了時）にリストのItemsSourceをクリアし、不要なアイコンリソースを解放するよう配慮されています。

## 4. クラス構造とデータフロー

1. アプリケーション起動時、`MainWindow` が非表示状態で生成され、`KeyboardHook` の初期化とシステムトレイへの登録が行われます。
2. ユーザーがWinキーを長押しする。
3. `KeyboardHook` が長押しを検知し、`MainWindow` へイベントを通知する。
4. `MainWindow` は `TaskbarReader.GetTaskbarApps()` を呼び出す。
5. `TaskbarReader` はUI Automationでタスクバーを解析し、`IconExtractor` を用いてアイコンを取得。`TaskbarApp` のリストを返す。
6. `MainWindow` はリストをUIにバインドし、フェードインアニメーションを再生して表示する。
7. ユーザーがWinキーを離すか、別のキーを入力する。
8. `KeyboardHook` がリリースイベントを通知し、`MainWindow` がフェードアウトアニメーションを再生して非表示状態に戻る。

## 5. データの永続化

アプリケーションの各種設定は、Windowsレジストリを利用して保存されています。
- **スタートアップ設定**: `HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Run` に自身のパスを書き込むことで、Windows起動時の自動実行を実現しています。

## 6. 特記事項・今後の拡張性

- **UI Automationへの依存**: タスクバーの構造はWindowsのバージョン（特にWindows 10と11）によって異なる場合があります。現状はUI Automationを用いて要素を動的に検索しているため一定の耐性がありますが、OSのメジャーアップデート時に構造が変わった場合は `TaskbarReader` の調整が必要になる可能性があります。
- **パフォーマンスと非同期処理**: アイコンの取得ロジックは負荷がかかる場合があるため、パフォーマンスチューニングや非同期処理（`async/await`）の導入により、UIのレスポンスやフェードインの滑らかさをさらに向上させる余地があります。
